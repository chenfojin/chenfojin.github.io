storcli

StorCLI 是由 Broadcom（之前的 LSI 和 Avago Technologies）开发的，专门用于管理和配置其 MegaRAID 控制器的命令行工具。

只有带有这些控制器的 RAID 卡才能使用 StorCLI。其他厂商的 RAID 卡可能需要使用其他的管理工具。例如，HP 的 Smart Array 控制器需要使用 HP 提供的 Smart Storage Administrator (SSA) 或者 Array Configuration Utility (ACU)，而 Dell 的 PERC 控制器需要使用 Dell 提供的 OpenManage Server Administrator (OMSA)。

安装storcli

官网下载地址

rpm -ivh storcli-007.1410.0000.0000-1.noarch.rpms

# 不管哪个目录rpm，最后安装完都在opt下，创建软链接方便使用
ln -s /opt/MegaRAID/storcli/storcli64 /bin/storcli
ln -s /opt/MegaRAID/storcli/storcli64 /sbin/storcli

日志收集

请把收集的信息（raid1.txt、raid2.txt、SN.txt、c0.txt、c1.txt、ROC.txt、Hard.txt）回传分析

1、storcli收集阵列日志

storcli /c0 show events file=raid1.txt                  # 收集日志raid1.txt，c0代表阵列卡0
storcli /c1 show events file=raid2.txt                  # 收集日志raid2.txt，c1代表阵列卡1
# 查看日志说明（得到的日志ctrl+F查找：）
Controller temperature threshold exceeded               # 温度高掉盘
0000 00                                                 # 有数据异常
Error                                                   # 有掉盘可能性
Fail                                                    # 查找阵列掉盘状态的变化

2、查看机器SN号

dmidecode | grep Ser >>SN.txt

3、掉盘故障处理

# 阵列下硬盘状态：
storcli /c0 show >>c0.txt
storcli /c1 show >>c1.txt

# 过滤raid卡温度:
storcli /call  show all | grep ROC>>ROC.txt

# 过滤硬盘温度--健康度:
storcli /call/eall/sall show all | grep -Ei '(Det|Error|Count)'>>Hard.txt

# 说明：
Medai Error Count=0                             # 0代表盘没问题，不为0如3代表硬盘问题
Predictive Failure Count=0                      # 0代表盘没问题，不为0如3代表硬盘问题
Detailed Information                            # 表示磁盘的插槽位置，Detailed Information
Drive Temperature                               # 当前硬盘温度
Medai Error Count                               # 表示磁盘可能错误，可能是磁盘有坏道，这个值不为0值得注意，数值越大，危险系数越高
Other Error Count                               # 表示磁盘可能存在松动，可能需要重新再插入
Predictive Failure Count                        # 预测性失败统计
Raw Size                                        # 硬盘大小
ROC temperature                                 # 控制芯片温度

基本语法

storcli /call/eall/sall show all

/c              # controller 控制器号
/e              # Enclosure  背板号，这个术语在计算机硬件中通常指的是一个物理结构，指安装硬盘的物理结构
/s              # slot/pd    物理硬盘的slot号
/v              # vd         虚拟硬盘，virtual drive
/f              # foreign    外部配置
/d              # disk group 磁盘组，一个dg可以有多个vd

查看raid控制器

storcli show              # 查看raid控制器摘要信息
storcli show all          # 查看raid控制器详细信息
storcli help              # 查看帮助信息
storcli /c0 show alarm    # 查看蜂鸣器状态
storcli /c0 set alarm=on  # 启用蜂鸣器

物理磁盘

查看物理磁盘状态
# 查看所有控制器 所有背板 背板上的所有磁盘的详细信息
storcli /call/eall/sall show all

# 主要关注以下几项
Medai Error Count=0             # 0代表盘没问题，不为0如3代表硬盘问题
Predictive Failure Count=0      # 0代表盘没问题，不为0如3代表硬盘问题
Detailed Information            # 表示磁盘的插槽位置，Detailed Information
Drive Temperature               # 当前硬盘温度
Medai Error Count               # 表示磁盘可能错误，可能是磁盘有坏道，这个值不为0值得注意，数值越大，危险系数越高
Other Error Count               # 表示磁盘可能存在松动，可能需要重新再插入
Predictive Failure Count        # 预测性失败统计
Raw Size                        # 硬盘大小
ROC temperature                 # 控制芯片温度

设置磁盘状态
# 查看所有控制器 所有背板 背板上的所有磁盘的详细信息
storcli /call/eall/sall show all

# 设置控制器 0 背板252槽位号为1的磁盘状态为good
storcli /c0/e252/s1 set good

# 设置控制器 0 背板252槽位号为1的磁盘状态为offline
storcli /c0/e252/s1 set offline

# 设置控制器为0 背板为252槽位号为1的磁盘状态为online
storcli /c0/e252/s1 set online
磁盘初始化
# 查看正在初始化的磁盘
storcli /cx/ex/sx show initialization

# 磁盘开始初始化
storcli /cx/ex/sx start initialization

# 停止磁盘的初始化
storcli /cx/ex/sx stop initialization

raid操作

查看raid
# 查看控制器上的所有
storcli /call/vall show

# 查看具体的RAID虚拟设备号为1的详细信息
storcli /c0/v1 show all

新建raid
storcli /cx add vd r[0|1|5|6|00|10|50|60]
        [Size=<VD1_Sz>,<VD2_Sz>,..|remaining] [name=<VDNAME1>,..]
        drives=[e:]s|[e:]s-x|[e:]s-x,y [PDperArray=x][SED]
        [pdcache=on|off|default][pi][DimmerSwitch(ds)=default|automatic(auto)|
        none|maximum(max)|MaximumWithoutCaching(maxnocache)][WT|WB|AWB][nora|ra]
        [direct|cached] [cachevd] [unmap][Strip=<8|16|32|64|128|256|512|1024>]
         [AfterVd=X] [EmulationType=0|1|2] [Spares = [e:]s|[e:]s-x|[e:]s-x,y]
        [force][ExclusiveAccess] [Cbsize=0|1|2 Cbmode=0|1|2|3|4|7]

# 使用默认参数创建的RAID5 读策略为ra: Read ahead 写策略wt io策略为direct 条带大小为256
storcli /c0 add vd r5 size=all name=Araid5 drives=252:0-11

# 设置读写策略条带大小的RAID5
storcli /c0 add vd r5 size=all name=Araid5 drives=252:0-11 awb ra strip=1024
删除raid
# 查看对应的vd号
storcli /call/vall show

# 删除raid
storcli /c0/vall del

# 删除含有用户数据（mbr或者分区信息）的raid
storcli /cx/vx del force
raid缓存设置

WT：Write Through。在写入缓存的同时，也将数据写入到背后的存储设备。这种方式可以提高数据的安全性，因为即使缓存失效，数据已经被保存在存储设备上。但是，它的缺点是写入操作的速度会受到存储设备速度的限制。

WB：Write Back。先将数据写入到缓存中，当缓存满时或者有空闲的机会时，再将数据写入到背后的存储设备。这种方式的优点是写入操作的速度会非常快，因为它只受到缓存速度的限制。但是，它的缺点是如果缓存突然失效（例如，由于电源中断），那么尚未写入到存储设备的数据会丢失。

AWB：Adaptive Write Back。这是一种混合策略，它会根据实际的工作负载和情况自动选择使用 Write Through 还是 Write Back 策略。这种方式旨在平衡数据的安全性和写入性能==（一般为raid卡含有电池的情况下会开启写缓存）==。

RA：Read-Ahead。这是一种预读策略，当系统从硬盘读取数据时，会预先读取更多的数据到缓存中，预期这些数据会在接下来的操作中被使用。这种策略利用了程序访问数据的局部性原理，可以显著提高连续读取的性能，因为从缓存读取数据比直接从硬盘读取数据要快得多。

# 设置写入缓存前需要先设置io优先级为缓存
storcli /cx/vx set iopolicy=direct|cached

# 设置写入缓存语法
storcli /cx/vx set wrcache=wt|wb|awb

# 设置读缓存语法，nora不预读，ra预读
storcli /cx/vx set racache=nora|ra
raid初始化

RAID5（5块以上）和RAID6（6块以上）在进行快速初始化时会转到后台进行初始化。

# 查看快速初始化
storcli /cx/vx show init

# 开启快速初始化
storcli /cx/vx start init

# 开启完全初始化
storcli /cx/vx start init full

# 停止初始化
storcli /cx/vx stop init

# 查看后台初始化
storcli /cx/vx show bgi

# 停止初始化
Storcli /all/v1 stop bgi

# 挂起初始化
storcli /call/v1 suspend bgi
raid一致性校验

cc：consistency check 一致性校验

# 查看初始化
storcli /cx/vx show cc

# 开启初始化
storcli /cx/vx start cc

# 停止初始化
storcli /cx/vx stop cc

## LSIRAID卡自带的一致性校验
# 查看一致性校验执行的具体时间
storcli /cx show cc

# 设置卡自带的一致性校验方式顺序或者并发
storcli /cx set consistencycheck=seq|con

# 关闭卡自带的一致性校验
storcli /cx set cc=off
热备

查看磁盘状态：storcli /c0 /eall/sall show 磁盘状态为UGOOD（unconfig good）空闲盘，磁盘状态为DHS（detect hot spare）为专属热备，磁盘状态为GHS（global hot spare）为全局热备

# 获取dgs号，获取磁盘状态找到ugood的磁盘
storcli /call/vall show
storcli /c0 /eall/sall show

# 添加专属热备
storcli /c0/e252/s6 add hotsparedrive dgs=0

# 添加全局热备
storcli /c0/e252/s6 add hotsparedrive

# 删除热备
storcli /c0/ex/sx delete hotsparedrive

日志

# 清除所有日志事件
storcli /cx clear events

# 将日志信息保存到指定文件
storcli /cx show events file=/path/file

# 删除TTY（用于故障定位的固件输出信息） 日志
storcli /cx delete termlog

# 查看产生日志文件的历史信息
storcli /cx show eventloginfo

# 查看term log 日志配置或者日志信息
storcli /cx show termlog type=config|contents

异常ubad恢复

1、磁盘状态为UBad

storcli /cx/ex/sall show all

2、重新设置磁盘状态为good

storcli /c0/ex/sx set good
storcli /c1/e252/sall show

3、查看硬盘状态，确认磁盘是否有故障问题（0代表没有故障）

storcli /c1/e252/sall show all

4、导入磁盘，检查磁盘状态为重建Rbld

storcli /cx/fall import

5、检查磁盘状态Rbld，检查重建进度

storcli /cx/ex/sx show
storcli /c0/e217/sall show rebuild
