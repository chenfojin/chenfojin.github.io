关于IOPS

iops详解

inupt/output operations per second 每秒操作读写次数

有些硬件会因为其==队列深度增加而提升其性能==，这多半是因为硬盘处理队列及重新排序（reordering）的先进控制器逻辑的结果，此逻辑一般称为标记命令队列（TCQ）或原生指令排序（NCQ）。企业等级的SATA硬盘，例如Western Digital Raptor及希捷的Barracuda NL配合深队列可以提升性能到100%。==较常用在服务器的高端SCSI硬盘==，一般性能有更大的提升。

常见的IOPS平均值

设备 IOPS 接口
7200 rpm sata盘 75-100 SATA 3Gbit/s
10000 rpm sata盘 125-150 SATA 3Gbit/s
15000 rpm sas盘 175-210 SAS(串列SCSI)
SSD 8600 SATA 3Gbit/s
Corsair Force Series GT SSD 85000 SATA 6Gbit/s

参考维基

  • 寻道时间：Tseek是指将读写磁头移动至正确的磁道上所需要的时间。寻道时间越短，I/O操作越快，目前磁盘的平均寻道时间一般在3－15ms。
  • 旋转延迟：Trotation是指盘片旋转将请求数据所在扇区移至读写磁头下方所需要的时间。旋转延迟取决于磁盘转速，通常使用磁盘旋转一周所需时间的1/2表示。比如，7200 rpm的磁盘平均旋转延迟大约为60*1000/7200/2 = 4.17ms，而转速为15000 rpm的磁盘其平均旋转延迟为2ms。
  • 数据传输时间：Ttransfer是指完成传输所请求的数据所需要的时间，它取决于数据传输率，其值等于数据大小除以数据传输率。目前IDE/ATA能达到133MB/s，SATA II可达到300MB/s的接口数据传输率，数据传输时间通常远小于前两部分消耗时间。简单计算时可忽略。

最大IOPS的理论计算方法：
IOPS = 1000 ms/ (寻道时间 + 旋转延迟)。可以忽略数据传输时间。

fio命令

fio参考
iodepth深度理解

# 测试4k随机读取
fio --filename=device_name --direct=1 --rw=randread --bs=4k --ioengine=libaio --iodepth=256 --runtime=120 --numjobs=4 --time_based --group_reporting --name=iops-test-job --eta-newline=1 --readonly
# 测试4k随机读写
fio --filename=/mount_point/file --size=500GB --direct=1 --rw=randrw --bs=4k --ioengine=libaio --iodepth=256 --runtime=120 --numjobs=4 --time_based --group_reporting --name=iops-test-job --eta-newline=1

配置文件

filename			# 文件（设备）名称
directory			# filename的路径前缀
name				# job的名字
direct				# bool类型，true（1），表示不适用io buffer
ioengine			# I/O引擎，默认值sync同步阻塞I/O（优点是易于理解和实现，缺点是性能较低，因为不支持并发操作），libaio是异步I/O（高性能支持并发）
iodepth				# 使用异步I/O时用到的，direct必须为1，buffer不支持异步
rw					# I/O模式，随机读写randread顺序读写read等
bs					# I/O block块大小，默认4k
size				# 指定job处理的文件的大小
numjobs				# 指定job的克隆数（线程），测试并发工作的负载表现
time_based			# 如果runtime指定时间内文件读写完成，将重复直到runtime时间结束
runtime				# 指定时间停止进程，单位s
group_reporting		# 制定了numjobs时，输出结果按组显示
stonewall			# 启用多个作业时，在启动作业前等待先前一个作业退出
readonly			# 启用只读安全检查，防止任何写入操作
allow_mounted_write	# 是否允许写操作，该测试将销毁已挂载文件系统上的数据默认false

example

创建配置文件fio.conf，运行fio fio.conf ；

[global]
ioengine=libaio
direct=1
iodepth=256
runtime=120
numjobs=1
filename=/home/test.db
size=1G
group_reporting
time_based
							
[4k_randread]
stonewall
bs=4k
rw=randread

[600k_seqwrite]
stonewall
bs=600k
rw=write
